PASSO 1: Configurar Supabase (5 min)
────────────────────────────────────

1. Abra https://app.supabase.com
2. Selecione projeto: lmedukids
3. Vá para Settings → API
4. Copie 2 chaves:
   • Anon (public) key
   • Service Role (secret) key

5. Cole em .env.supabase:
   
   SUPABASE_URL=https://ndmccxupbrplljalqegh.supabase.co
   SUPABASE_ANON_KEY=eyJhbGc... [copie aqui]
   SUPABASE_SERVICE_KEY=eyJhbGc... [copie aqui]

PASSO 2: Criar Tabelas (5 min)
───────────────────────────────

1. No Supabase Dashboard: SQL Editor
2. Clique "New Query"
3. Copie TODO o conteúdo de: database/schema.sql
4. Cole no editor
5. Clique "Run"

✅ Você verá 7 tabelas criadas automaticamente

PASSO 3: Criar Storage (5 min)
────────────────────────────────

1. Supabase Dashboard → Storage
2. Clique "New Bucket" 3x:

   Bucket 1: user-avatars
   Privacy: Public
   
   Bucket 2: exercise-images
   Privacy: Private
   
   Bucket 3: teacher-content
   Privacy: Private

PASSO 4: Ativar Autenticação (2 min)
────────────────────────────────────

1. Supabase Dashboard → Authentication → Providers
2. Email/Password → Habilitar ✓
3. (Opcional) Google, GitHub, etc

🔐 SEGURANÇA
═══════════════════════════════════════════════════════════════════════════════

✅ Row Level Security (RLS)
   • Alunos só veem seus próprios dados
   • Professores só veem sua própria turma
   • Super seguro a nível de banco

✅ Autenticação
   • JWT tokens gerenciados pelo Supabase
   • Refresh tokens automáticos
   • Session persistence

✅ Storage Privado
   • Imagens privadas por usuário
   • Acesso controlado por Auth

✅ Variáveis de Ambiente
   • ANON_KEY é segura (pública)
   • SERVICE_KEY nunca é commitada
   • Diferentes keys para cada ambiente

🧪 COMO TESTAR LOCALMENTE
═══════════════════════════════════════════════════════════════════════════════

1. Guarde as API keys em .env.local

   // Signup
   import { signUpUser } from './supabase.js';
   await signUpUser('teste@example.com', 'senha123', 'João', 'student', 5);

   // Login
   import { loginUser } from './supabase.js';
   await loginUser('teste@example.com', 'senha123');

   // Usuário atual
   import { getCurrentUser } from './supabase.js';
   const { user, profile } = await getCurrentUser();
   console.log(profile);

4. Teste backend Vercel:

   npm start (na pasta backend)
   curl http://localhost:3456/health

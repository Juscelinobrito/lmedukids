🎯 FLUXO DE DEPLOY RECOMENDADO
═══════════════════════════════════════════════════════════════════════════════

PASSO 1: Conectar Repositório ao Vercel (5 min)
────────────────────────────────────────────────
1. Vá para https://vercel.com/new
2. Clique "Import Git Repository"
3. Cole: https://github.com/Juscelinobrito/lmedukids
4. Clique "Import"

PASSO 2: Configurar Environment Variables (5 min)
────────────────────────────────────────────────
Na tela de configuração do Vercel, adicione:

   OPENAI_API_KEY
   Valor: sk-proj-[sua-chave-real]
   Ambientes: Production ✓ Preview ✓ Development ✓

   ENABLE_PRACTICE_MODE
   Valor: false
   Ambientes: Production ☐ Preview ✓ Development ☐
   (Só em Preview = staging)

   ENABLE_GRADING
   Valor: false
   Ambientes: Production ☐ Preview ✓ Development ☐
   (Só em Preview = staging)

   NODE_ENV
   Valor: production
   Ambientes: Production ✓ Preview ✓ Development ✓

   ENVIRONMENT
   Valor: staging
   Ambientes: Production ☐ Preview ✓ Development ☐

5. Clique "Deploy"

PASSO 3: Aguardar Build (2-3 min)
────────────────────────────────
Vercel compilará e fará deploy automático.
Você receberá uma URL: https://lmedukids-[hash].vercel.app

PASSO 4: Validar Deploy (2 min)
────────────────────────────
No navegador, abra a URL e:

✓ Verifique upload e análise funcionam
✗ Verifique que o tab "🎯 Praticar" NÃO aparece (como esperado!)
✓ Abra o console e execute:
  fetch('/api/features').then(r=>r.json()).then(console.log)
  
  Deve mostrar:
  {
    "environment": "staging",
    "features": {
      "PRACTICE_MODE": false,
      "GRADING": false,
      "CACHE": true,
      "RATE_LIMIT": true
    }
  }

📝 PROCEDIMENTO: GIT PUSH → DEPLOY AUTOMÁTICO
═══════════════════════════════════════════════════════════════════════════════

1. Você faz mudanças locais
   $ npm start
   $ # testar modo prática, exercícios, etc

2. Você commita e faz push
   $ git add -A
   $ git commit -m "feat: melhorias no modo prática"
   $ git push origin develop

3. GitHub envia webhook ao Vercel
   (automático, você não faz nada)

4. Vercel detecta novo push
   • Lê variáveis de ambiente (ENABLE_PRACTICE_MODE=false para Preview)
   • Build automático
   • Deploy em nova URL: https://lmedukids-pr-123.vercel.app

5. Preview URL fica pronta
   • Você compartilha com beta testers
   • Eles testam sem ver as novas features (como esperado!)
   • Você coleta feedback

🔧 COMO ATIVAR FEATURES EM STAGING (SE NECESSÁRIO)
═══════════════════════════════════════════════════════════════════════════════

Se você precisar testar as NOVAS features (Prática + Grading) em staging:

1. Vá para Vercel Dashboard
2. Settings → Environment Variables
3. Encontre ENABLE_PRACTICE_MODE
4. Mude o valor de "false" para "true"
5. Redeployar (clique "Redeploy")

Agora a Preview URL (staging) mostrará as novas features!

Para esconder novamente: mude para "false" e redeployar.

📞 SUPORTE / DÚVIDAS
═══════════════════════════════════════════════════════════════════════════════

Problema: Deploy falha no Vercel
Solução:
  1. Verificar logs em Vercel Dashboard → Deployments
  2. Verificar se backend/package.json tem todas as dependências
  3. Verificar se OPENAI_API_KEY está configurada

Problema: Features aparecem em staging (deveriam estar ocultas)
Solução:
  1. Verificar ENABLE_PRACTICE_MODE no console: fetch('/api/features')
  2. Se for true, significa Vercel env vars não foram aplicadas
  3. Redeployer no Vercel Dashboard

Problema: URL local (localhost:3456) não funciona após mudança
Solução:
  1. npm install (renovar dependências)
  2. npm start
  3. Limpar cache do browser (Ctrl+Shift+Del)
